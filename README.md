# 双人漫游簿

一个紫色像素风、情侣主题的手机旅行收藏与路线规划网页原型。直接打开 `index.html` 即可浏览；通过本地网页服务打开时，还支持安装到手机桌面与离线缓存。

## 当前可体验

- 顶部城市菜单、添加城市与城市切换
- 小红书 / 抖音分享链接识别流程演示
- 地点、拍照机位、来源封面归档到“城市口袋”
- 输入旅行天数与住宿地点
- 根据地点坐标聚类、从住宿点出发做近邻路线排序
- 每日路线、距离、餐厅与拍照建议融合
- 推荐榜收藏后自动参与路线计算
- 原创可替换的双人像素角色组件
- 生成并下载 1080 × 1440 攻略 PNG
- 浏览器本地保存、每天 10:00 刷新状态与离线访问

## 接入真实服务时需要的接口

1. `POST /api/import-link`
   - 输入：分享链接、城市
   - 服务端读取分享落地页或平台合规接口，保存封面与正文图片
   - OCR + 地点实体识别，返回地点、机位、营业时间、经纬度和可信度

2. `POST /api/plan`
   - 输入：住宿经纬度、天数、收藏地点、出行偏好
   - 接入高德 / 腾讯地图路线矩阵，按真实通勤时间聚类并排序
   - 融合天气、闭馆日、预约时间和用餐时段

3. `GET /api/city-ranking?city=成都`
   - 聚合有授权的数据源、地图 POI 热度、点评与编辑推荐
   - 对来源、抓取时间与广告内容做标注

4. 每天 10:00 的定时任务
   - 服务端定时读取前一天新增链接
   - 重新识别、去重、生成路线与手绘图
   - 通过网页推送或微信服务通知用户

> 注意：小红书和抖音页面通常有登录、访问频率与内容版权限制。生产环境应优先使用平台允许的分享元数据或用户主动上传的截图，不应在纯前端保存登录凭证或绕过平台限制。
